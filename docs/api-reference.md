# API Reference

This reference documentation lists the core Kotlin and C APIs available in CBindingKMP.

## Kotlin API

### `com.abyxcz.cbindingkmp.shared.NativeLoader`

Helper object to manage the loading of the native shared library.

-   `fun load()`: 
    -   Loads the "mylib" shared library on Android and JVM.
    -   No-op on iOS and other native targets.
    -   **Usage**: Call this before any native function access.

---

### `com.abyxcz.cbindingkmp.shared.generated.GeneratedNativeKt`

This is a generated file containing the JNI bindings for your C library. By default, these functions are `internal`.

-   `external fun <function_name>JNI(...)`:
    -   Direct call to the JNI wrapper for the corresponding C function.
    -   Example: `external fun add_numbersJNI(a: Int, b: Int): Int`.

---

### `com.abyxcz.cbindingkmp.shared.Calculator` (Example)

A sample high-level wrapper that demonstrates how to bridge platform-specific calls.

-   `fun add(a: Int, b: Int): Int`: Bridges between `add_numbersJNI` (Android/JVM) and direct `add_numbers` (iOS).

---

## C API

### `native/c/mylib.h`

The primary interface for your native C code.

-   `int add_numbers(int a, int b)`: Adds two integers.
-   `int multiply(int a, int b)`: Multiplies two integers.

---

## Generated C API

### `shared/build/generated/jni/jni_gen_bridge.c`

This file contains the JNI wrappers generated by the plugin. These are not intended to be called directly by developers but are used by the JVM/Android runtime.

-   `JNIEXPORT <jni_type> JNICALL Java_com_abyxcz_cbindingkmp_shared_generated_GeneratedNativeKt_<function_name>JNI(...)`:
    -   Standard JNI wrapper that delegates the call to the actual C implementation.
